# 博客部署说明

## 文件位置
- 完整项目：`C:\tmp\my-blog`
- ZIP打包：`C:\tmp\my-blog.zip`

## 项目结构
```
my-blog/
├── .github/workflows/deploy.yml  # GitHub Actions 自动部署
├── .gitignore
├── astro.config.mjs              # Astro 配置
├── package.json
├── tailwind.config.mjs           # Tailwind 配置
├── tsconfig.json
└── src/
    ├── layouts/
    │   └── Layout.astro           # 主布局（侧边栏+分类）
    ├── pages/
    │   ├── index.astro           # 首页
    │   ├── about.astro           # 关于页
    │   └── category/
    │       └── [slug].astro      # 分类页（动态路由）
    └── styles/
```

## 本地运行步骤

1. 解压 ZIP 到本地文件夹（如 `my-blog`）

2. 安装依赖：
```bash
cd my-blog
pnpm install
```

3. 本地预览：
```bash
pnpm dev
```
访问 http://localhost:4321

4. 构建：
```bash
pnpm build
```

## GitHub 部署步骤

1. 登录 GitHub：https://github.com/jarenivss-a11y/my-blog

2. 由于网络限制，你需要本地操作：
   - 在本地克隆仓库（如果能访问）
   - 或者手动上传文件到 GitHub

3. 开启 GitHub Pages：
   - 进入 Settings → Pages
   - Source 选择 "GitHub Actions"
   - 保存

4. 等待 GitHub Actions 自动构建（约2-3分钟）

5. 访问博客：https://jarenivss-a11y.github.io/my-blog

## 添加/删除分类

编辑 `src/layouts/Layout.astro` 中的 categories 数组：

```javascript
const categories = [
  { name: '日常生活', slug: 'daily', icon: '🏠' },
  { name: '音乐分析', slug: 'music', icon: '🎵' },
  // 添加新分类...
  // 删除分类：直接删除这一行
];
```

同时编辑 `src/pages/category/[slug].astro` 中的 getStaticPaths。

## 常见问题

Q: 构建失败？
A: 确保 Node.js >= 18，运行 `pnpm install` 后再试。

Q: 分类不显示？
A: 检查 Layout.astro 和 [slug].astro 两个文件的 categories 是否一致。

Q: 如何添加文章？
A: 在 src/content/blog/ 下创建 .md 文件，Frontmatter 示例：
```yaml
---
title: 我的文章
published: 2026-06-24
category: daily
---
文章内容...
```
